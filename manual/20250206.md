## 기본 자료형과 연산자
: JAVA와 겹치는 부분이 있음.

### 자료형
: 숫자, 텍스트, 참/거짓, 복합 등 다양한 종류의 데이터가 사용되는데, 각 데이터마다 필요로 하는 메모리 용량에도 차이가 있다. 다만, JS는 동적 자료형을 지니고 있어 엄격하지는 않다.
#### boolean
- 우리가 아는 true/false
#### typeof
- typeof 연산자는 어떤 데이터나 자료형을 붙여도 그 데이터의 자료형이 무엇인지 알려주는 연산자로, boolean 결과값이 반환된다.

#### number
- 숫자로 된 자료형. JS는 정수, 실수형을 구분하지 않고 하나의 숫자 자료형으로 간주한다.
- 숫자 리터럴끼리의 연산 가능
- 숫자 리터럴과 변수 / 상수와의 연산도 가능하다.

리터럴(literal)이란? : 변수에 저장되어 있지 않은 순수 데이터
```javascript
let a = 1;
console.log(a+2);
//여기서 2가 리터럴에 해당한다.
```

#### string
C언어나 자바에서는 클래스이기 때문에 대문자로 시작하나, JS에서는 그냥 자료형이라 소문자로 시작한다.\
자바에서는 문자열 자료형을 표시할 때 작은따옴표, 큰따옴표를 구분하지만, JS에서는 구분하지 않는다. 한글/영어/이모지/유니코드로 표기 가능한 모든 문자가 들어갈 수 있다.

- 문자열 자료형도 덧셈 연산이 가능하다. +는 두 문자열을 이어붙이는 역할을 한다.
- 문자열을 변수로 사용하는 것도 당연히 가능하다.

#### undefined vs null
1. undefined: 상자에 무엇이 들어있는지 명시되지 않은 상태. 선언만 되어 있고, 초기화가 된지 않은 상태.
2. null: 주머니가 비어있다고 지정된 상태.

- 값을 아예 모르는 상태와 비어있는 상태의 차이.
```javascript
let x;
console.log(x) //undefined
let y = null;
console.log(y) //null
```

JS 상에서의 부실 공사 관련 이슈 : 웹 사이트를 만들 때 사용하는 언어이다보니, 오류 발생마다 다 뜯어고치게 되면 이전 버전을 사용하는 웹사이트들에게서 연쇄적으로 문제가 발생하기 때문에 다 뜯어고치기 보다는 더 나은 기능을 지닌 새로운 명령어를 만들어냄으로써 추후 사용자들이 새 기능을 이용하도록 장려함\
(예를 들어 var -> let과 같은 현상상)

### 연산자
- typeof / ! / % / 사칙연산자 이외의 추가적인 연산자.
#### 비교연산자
- 좌우에 있는 데이터를 비교해서 'boolean'으로 반환하는 연산자.
##### 종류
1. == : **자료형에 관계없이** 같은 값.
2. === : **자료형이랑 값 모두** 같은 값.
3. != : ==의 반대
4. !== : ===의 반대
5. \> / >= / < / <= : 우리가 아는 그 부등호.

#### 부수 효과를 일으키는 연산자
변수의 값만 비교하는 것이 아니라, 변수의 값을 바꾸는 연산자들
##### 종류
1. ++ 2. -- 3. 변수명++ 4. ++변수명
자바와 동일하다.

#### 할당 산술 연산자
사용되는 변수에 새로이 연산된 값을 할당하는 연산자
##### 종류
.1. x+=y 2. x-=y 3. x*=y 4. x/=y 5. x%=y
마찬가지로 자바와 동일하다.

\
6. x**=y -> 자바에서의 `Math.pow(x,y)`와 동일. 파이썬에서는 이렇게 썼던 것으로 기억하고 있다.

문자열 자료형에서도 할당 산술 연산자 사용이 가능한데, +=만 사용할 수 있다.

#### boolean 관련 연산자
##### 종류
1. && : AND : 곱연산자
2. || : OR : 합연산자
3. ! : NOT : 부정

#### 삼항연산자
##### 형식 : (조건) ? 참일 때 : 거짓일 때

## 객체와 배열
타 프로그래밍 언어에서는 배열이 좀 더 단순한 형태이기 때문에 배열만 먼저 배우는 경우가 많지만, 자밧크립트의 경우에는 배열이 객체의 한 종류에 포함되어 있다.
### 객체
- 기본 자료형(원시 자료형(primitive type)) : 숫자, 문자열, boolean -> 각 자료형이 단 하나의 데이터만 가지고 있는 형태.
- 복합 자료형 : 데이터 여러 개를 함께 묶어서 사용한다. -> 원시 자료형 이외의 모든 자료형
  - 프로퍼티(property) : 키-값 쌍으로 이루어진 형태를 의미한다. 객체는 프로퍼티로 구성되어 있다. (자바에서는 Entry라는 용어를 사용한다.)
```javascript
  //객체, 프로퍼티 관련
  const person01 = {
  name: "김철수",
  age: 25,
  address: "부산광역시 연제구",
  married: false
  }
  //key에 큰따옴표가 없기 때문에 JSON이 아니다. JSON하고 비슷하게 생긴 것 뿐이다.

  console.log(typeof(person01));
  console.log(person01);
  /*
  {
      "name": "김철수",
      "age": 25,
      "address": "부산광역시 연제구",
      "married": false
  }
  개발자 도구에 우클릭해서 copy object를 클릭하니, JSON 형태로 복사되었다.
  */

  //객체 접근 방법 -> Java와 동일
  //1. 인스턴스 변수처럼 접근하는 방법.
  console.log(person01.name);
  //2. Java에서의 Map 자료형과 같은 방식.
  console.log(person01["name"]);
  console.log(person01.age);
  console.log(person01.address);
  console.log(person01.married);
```
["키 이름"]으로 객체를 찾는 이유? :
프로그래밍을 하다보면 코드를 작성할 때, 어떤 키 값이 들어올지 개발 중에는 모르는 경우가 많다. 예를 들어, 프로그램 실행 중에 사용자 입력 등과 같은 여러 조건에 따라 객체로부터 값을 얻어내야하는 경우(자바의 `Scanner.nextLine()`을 통해 사용자마다 입력값이 다를 수 있는 경우와 같이), 코드를 작성하는 시점에서는 키 값이 어떤 식으로 사용될지 대부분 알기 때문에 주로 전자와 같은 방식으로 작성한다.

단, 상황에 따라 후자의 방식처럼 `person["name"]`과 같은 형태로도 키 값을 찾는 경우가 있다.

기존 프로퍼티 수정 방식
```javascript
person01.age++;
person01["job"]="teacher";
console.log(person01.age,person01.job);
```

const로 지정된 상수는 어떻게 수정할까? 분명히 person01을 선언 시에는 const로 선언했는데...\
-> 해당 경우에서 person01은 객체, 즉 다수의 자료를 관리하는 자료형이었다. 따라서, person01을 수정한 것이 아니라, person01 내에 있는 key들 중 일부의 value를 수정한 것이기 때문에 오류가 발생하지 않았다.

```javascript
person01={} //const로 선언한 객체에, 빈 값들을 재대입
//Assignment to constant variable 오류 발생!
```
